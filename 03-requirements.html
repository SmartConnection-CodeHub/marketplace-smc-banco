02Funcionales · Inventario & Catálogo

FR-INV-01P1 · El sistema debe mantener UN inventario físico maestro por tenant · con stock real-time.
FR-INV-02P1 · El sistema debe permitir SKU master con: código interno, EAN/UPC, dimensiones, peso, categorías, variantes (talla/color).
FR-INV-03P1 · El sistema debe descontar stock al confirmar venta en CUALQUIER canal · sincronización inmediata cross-canal.
FR-INV-04P1 · El sistema debe permitir ajustes manuales con motivo (merma, daño, recuento, devolución).
FR-INV-05P2 · El sistema debe soportar reservas (stock asignado a orden pendiente vs físico disponible).
FR-INV-06P2 · El sistema debe asignar ubicaciones físicas (rack/pasillo/estante/bin) a cada SKU.
FR-INV-07P3 · El sistema debe soportar múltiples bodegas (Stgo + regiones) con stock por bodega.
FR-INV-08P3 · El sistema debe generar etiquetas barcode imprimibles (Code 128, QR).
Marketplace SMC · Blueprint v1.004

03Funcionales · Ventas Multi-Canal

FR-CH-01P1 · El sistema debe publicar 1 SKU en N canales con 1 click (Channel Adapter Pattern).
FR-CH-02P1 · El sistema debe aplicar pricing diferenciado por canal (landed + markup + comisión + IVA).
FR-CH-03P1 · Cada canal debe implementar interfaz CanalAdapter con métodos: publish, syncStock, fetchOrders, fulfillOrder, syncPrice.
FR-CH-04P1 · El sistema debe recibir webhooks de canales externos (MeLi, Falabella futuro) con HMAC validation.
FR-CH-05P1 · El sistema debe registrar TODA orden en tabla común orders con columna channel (no tablas separadas).
FR-CH-06P2 · El sistema debe soportar quote builder B2B (selección SKUs → PDF cotización → aprobación email).
FR-CH-07P2 · El sistema debe monitorear API Mercado Público diariamente · filtrar licitaciones por keywords del operador.
FR-CH-08P2 · El sistema debe permitir storefront D2C con checkout propio (carrito + pasarela).
FR-CH-09P3 · El sistema debe soportar canal "manual" (Cencosud · scrape semanal + upload CSV).
Marketplace SMC · Blueprint v1.005

04Funcionales · Sourcing & Supply Chain

FR-SUP-01P2 · El sistema debe gestionar proveedores (Nexport · futuros) con CRM ligero.
FR-SUP-02P2 · El sistema debe permitir órdenes de compra (PO) con términos de pago (50/50 · 100% adelanto · etc).
FR-SUP-03P2 · El sistema debe trackear shipments con ETA · alertas pre-arribo.
FR-SUP-04P2 · El sistema debe registrar recepción con match contra PO · detectar discrepancias (cantidad/daños).
FR-SUP-05P1 · El sistema debe calcular landed cost por SKU (FOB + flete + IVA + transporte interno).
FR-SUP-06P1 · El sistema debe integrarse con couriers (Chilexpress · Starken · Bluexpress) para generar etiquetas envío.
FR-SUP-07P2 · El sistema debe permitir reglas de despacho ("MeLi → Chilexpress" · "B2B → Starken").
FR-SUP-08P3 · El sistema debe gestionar devoluciones (RMA) con flujo aprobación + NC.
Marketplace SMC · Blueprint v1.006

05Funcionales · Finanzas & Contabilidad

FR-FIN-01P1 · El sistema debe emitir DTE (Factura 33, Boleta 39, NC 61, ND 56, Factura Exenta 34) vía Open Factura o SII MIPYME.
FR-FIN-02P1 · El sistema debe generar asientos contables automáticos por cada evento (venta · compra · pago · devolución).
FR-FIN-03P1 · El sistema debe mantener cuentas por cobrar (CxC) y por pagar (CxP) con vencimientos.
FR-FIN-04P1 · El sistema debe alertar facturas vencidas: 5 días antes · al día · +7 · +15 · +30.
FR-FIN-05P2 · El sistema debe calcular margen real por SKU = Precio − Landed − Comisión canal − Logística − IVA débito.
FR-FIN-06P2 · El sistema debe pre-generar resumen para F29 mensual (ventas afectas + IVA débito + IVA crédito).
FR-FIN-07P2 · El sistema debe forecastear flujo de caja a 30/60/90 días.
FR-FIN-08P3 · El sistema debe sugerir precio óptimo por canal vía IA (vs competencia · margen target).
Marketplace SMC · Blueprint v1.007

06Funcionales · IA & Analytics

FR-AI-01P2 · El sistema debe ofrecer chat IA contextual sobre la data del operador (Cerebro IA).
FR-AI-02P2 · El sistema debe generar insights semanales auto-reporte (lunes 9am).
FR-AI-03P3 · El sistema debe sugerir SKUs a re-comprar según rotación + lead time proveedor.
FR-AN-01P2 · El sistema debe mostrar dashboard consolidado: revenue · margen · órdenes · ticket promedio · split por canal.
FR-AN-02P2 · El sistema debe permitir drill-down por canal · marca (tenant) · vertical · producto · período.
FR-AN-03P3 · El sistema debe analizar rotación por SKU · top performers · slow movers.
FR-AN-04P3 · El sistema debe medir ROI por canal de venta y por proveedor.

08No funcionales · Seguridad

NFR-S-01P1 · El sistema debe implementar Row Level Security (RLS) en TODAS las tablas con tenant_id.
NFR-S-02P1 · Las credenciales OAuth de canales externos deben guardarse cifradas at rest (AES-256).
NFR-S-03P1 · Todas las APIs internas deben autenticar con JWT o sesión httpOnly.
NFR-S-04P1 · Los webhooks externos deben validar firma HMAC obligatorio.
NFR-S-05P1 · Los secrets (API keys, DB passwords) NUNCA en código · solo env vars + secret manager.
NFR-S-06P2 · El sistema debe registrar audit log de toda mutación financiera (asientos, DTE, pagos).
NFR-S-07P2 · El sistema debe rotar tokens MeLi/MP automáticamente (cron) antes de expirar.
NFR-S-08P2 · El sistema debe ofrecer 2FA para login operador.
NFR-S-09P3 · El sistema debe permitir export GDPR-style de toda la data de un cliente (B2B/D2C).
Marketplace SMC · Blueprint v1.010

09No funcionales · Escalabilidad

NFR-SC-01P1 · La arquitectura debe ser multi-tenant nativo · soportar N tenants en misma BD con aislamiento RLS.
NFR-SC-02P1 · Cada canal debe ser un adapter independiente · agregar canal nuevo no debe requerir tocar core.
NFR-SC-03P2 · El sistema debe soportar 100 órdenes/hora por tenant sin degradación (MVP).
NFR-SC-04P2 · El sistema debe soportar 100K SKUs por tenant (catálogo grande).
NFR-SC-05P3 · El sistema debe soportar 1.000 órdenes/hora por tenant (escalado fase 3).
NFR-SC-06P3 · El sistema debe ser desplegable en regiones distintas para latencia local.
NFR-SC-07P2 · La BD debe estar en Supabase (Postgres) · permite escalar vertical hasta 10K órdenes/hora antes de necesitar partition.
Marketplace SMC · Blueprint v1.011

10No funcionales · Compliance Chile

NFR-C-01P1 · El sistema debe respetar Ley 19.628 (protección datos personales Chile).
NFR-C-02P1 · El sistema debe emitir DTE válidos SII (estructura XML según normativa).
NFR-C-03P1 · El sistema debe mantener libro IVA digital (ventas + compras) según Resolución SII.
NFR-C-04P2 · El sistema debe permitir inventario anual (Resolución 73 SII).
NFR-C-05P2 · El sistema debe alertar vencimientos: F30 · F30-1 · garantías bancarias · vigencia ChileProveedores.
NFR-C-06P2 · El sistema debe soportar Régimen 14D N°3 Pro-PYME General (cálculo automático).
NFR-C-07P3 · El sistema debe pre-generar F22 anual con saldos contables del año.
NFR-C-08P3 · El sistema debe gestionar accesos auditados (quién vio qué · quién modificó qué).
