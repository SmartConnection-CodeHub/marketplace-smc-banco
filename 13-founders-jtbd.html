02Por qué este framework aplica aún sin clientes externos

Aunque NO vendamos a terceros · los founders contratan al sistema para hacer trabajos específicos. JTBD sirve para validar que el sistema entrega valor REAL a quienes lo usan (los 3) · y para detectar features que NO usamos = candidatos a eliminar.

04Lo que NO contrata el sistema (descartes explícitos)

  • NO contratamos al sistema para "generar leads" (no vendemos SaaS)
  • NO contratamos al sistema para "atención cliente externo escalada" (no tenemos clientes externos)
  • NO contratamos al sistema para "billing recurring" (no hay suscripciones)
  • NO contratamos al sistema para "white-label" (es para nosotros)
Banco SMC · Founders Operation06

05Por qué validar JTBD trimestralmente

Cada trimestre revisamos:

  • ¿Qué jobs declarados al inicio ya NO usamos? → candidatos a deprecar
  • ¿Qué jobs nuevos emergieron? → priorizar en roadmap
  • ¿Algún job lo hacemos manual aunque el sistema "lo cubre"? → bug o UX mala
