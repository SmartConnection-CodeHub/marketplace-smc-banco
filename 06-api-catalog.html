01Convenciones REST + versionado

  • Base URL: https://api.marketplace.smconnection.cl/v1
  • Auth: header Authorization: Bearer <jwt>
  • Tenant: header X-Tenant-Id: <uuid> · resuelve scope automático con RLS
  • Content-Type: application/json
  • Versionado: prefijo /v1 · cambios breaking suben a /v2
  • Paginación: ?page=1&per_page=50 · headers X-Total-Count
  • Errores: estructura estándar {error: {code, message, details}}
  • Status codes: 200 OK · 201 Created · 204 No Content · 400 Validation · 401 Auth · 403 Forbidden · 404 NotFound · 409 Conflict · 422 BusinessRule · 500 Error

02Channel Adapter Pattern · la clave

Esta es la pieza arquitectónica más importante del Hub. Cada canal externo (MeLi · MP · D2C · B2B · Falabella futuro) implementa la misma interfaz. El core nunca sabe qué canal es · solo llama métodos estándar.

Interfaz que cada canal debe implementar

// TypeScript · interfaz pura · cada canal la implementa interface ChannelAdapter { // Metadata type: 'meli' | 'mp' | 'd2c' | 'b2b' | 'falabella' | 'cencosud' name: string // === PUBLICAR === publish(product: Product, price: Price): Promise<ListingResult> unpublish(listing: Listing): Promise<void> updatePrice(listing: Listing, newPrice: Price): Promise<void> // === STOCK === syncStock(listing: Listing, qty: number): Promise<void> // === ÓRDENES === fetchOrders(since: Date): Promise<Order[]> fetchOrderById(externalId: string): Promise<Order> cancelOrder(order: Order, reason: string): Promise<void> // === FULFILLMENT === markShipped(order: Order, tracking: TrackingInfo): Promise<void> markDelivered(order: Order): Promise<void> // === HEALTH === healthCheck(): Promise<{ ok: boolean, latency_ms: number, last_error?: string }> refreshAuth(): Promise<void> }
🎯 Por qué es la pieza clave

Agregar Walmart Chile en 2027 = escribir 1 archivo WalmartAdapter.ts que implementa esta interfaz. Cero cambios en core, cero cambios en BD, cero cambios en UI. Plug-and-play real.
